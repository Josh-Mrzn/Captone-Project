# 🌾 AgriFair Backend Architecture (Layered MVC)

Welcome to the **AgriFair** Backend. This system is designed to enable **direct rice trading between farmers and buyers**, eliminating middlemen and ensuring fair pricing, transparency, and efficiency.

Built with a **4-Tier Layered MVC Architecture**, it ensures scalability, security (**DevSpec**), and real-time updates for transactions and order tracking.

---

## 📂 The Master Folder Structure

This structure enforces strict **Separation of Concerns** — keeping the system clean, scalable, and maintainable.

---

### 🌱 Core Layers
| Folder | Responsibility |
| :--- | :--- |
| **`routes/`** | The "Marketplace Map." Connects API endpoints (orders, farmers, buyers, listings) to Controllers. |
| **`controllers/`** | The "Handlers." Manage HTTP requests/responses with **no business logic**. |
| **`services/`** | The "Marketplace Brain." Handles pricing logic, order matching, inventory, and transaction rules. |
| **`repositories/`** | The "Data Access Layer." The **only** layer allowed to interact with the database (CRUD). |
| **`models/`** | The "Schemas." Defines data structures like Farmers, Buyers, Rice Listings, Orders. |

---

### 🛡️ Security & Operations (DevSpec)
| Folder | Responsibility |
| :--- | :--- |
| **`validators/`** | **The Gatekeeper.** Validates farmer listings, buyer orders, and user inputs (Zod/Joi). |
| **`middleware/`** | Global guards (Authentication, Rate Limiting, Error Handling). |
| **`errors/`** | Custom error handling for consistent API responses. |
| **`logs/`** | **Audit Trail.** Tracks transactions, errors, and system activity. |

---

### 🚜 Marketplace Features & Utilities
| Folder | Responsibility |
| :--- | :--- |
| **`sockets/`** | Real-time updates (order status, new listings, buyer-farmer notifications). |
| **`jobs/`** | Background tasks (price updates, order expiration, reporting). |
| **`scripts/`** | Data seeding (sample farmers, rice inventory). |
| **`storage/`** | Handles uploads (documents, certifications, proof of harvest). |
| **`utils/`** | Helpers (JWT auth, hashing, pricing calculators). |
| **`configs/`** | Configurations (DB connection, Redis, environment setup). |
| **`tests/`** | Unit and integration testing for reliability. |

## 🔄 The "AgriFair" Request Flow

1. **Client (Farmer/Buyer)** ➔ `routes/` (API endpoint access)
2. `routes/` ➔ `middleware/` (Authentication & security checks)
3. `middleware/` ➔ `validators/` (Input validation for listings/orders)
4. `validators/` ➔ `controllers/` (Request handling)
5. `controllers/` ➔ `services/` (Business logic: pricing, matching, order handling)
6. `services/` ➔ `repositories/` (Database operations)
7. `repositories/` ➔ `models/` (Data structure enforcement)
8. **Response** ➔ Sent back to Client  
   ⚡ Includes real-time updates via **Sockets** (order status, availability, transactions)
